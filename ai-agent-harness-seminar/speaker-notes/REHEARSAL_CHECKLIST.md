# Rehearsal Checklist

## 콘텐츠

- [ ] 75분 가정이 실제 일정과 일치한다.
- [ ] 사실·해석·비유·가정·권고가 구분되어 있다.
- [ ] Pending evidence가 화면에 확정 사실처럼 남지 않았다.
- [ ] 모든 수치의 단위·출처·계산을 다시 확인했다.
- [ ] “생산성 향상”을 근거 없이 말하지 않는다.
- [ ] Agent 비적용·보류·Do Nothing 대안을 설명한다.

## 화면과 발표 환경

- [ ] 실제 프로젝터, 해상도, 조도에서 뒤쪽 가독성을 확인했다.
- [ ] 색상 없이도 성공·실패·승인을 구분할 수 있다.
- [ ] Presenter 화면과 청중 화면 분리를 확인했다.
- [ ] Presenter Note와 키보드 이동이 동작한다.
- [ ] 오프라인 상태에서 설치된 프로젝트가 실행된다.
- [ ] 화면 캡처와 Trace에 민감정보가 없다.

## 데모

- [ ] fixture가 합성 데이터임을 화면에 표시한다.
- [ ] 입력 hash와 도구 버전을 기록한다.
- [ ] sampling frequency 누락 경로가 추정 없이 중단된다.
- [ ] Validator가 Agent와 독립적으로 실행된다.
- [ ] 승인 Gate와 비상 중단 방법을 실제로 확인했다.
- [ ] Live 제한 시간을 넘으면 대체 Trace로 전환한다.
- [ ] 대체 Trace를 녹화 실행이라고 명확히 말한다.
- [ ] 실제 진단 결과처럼 단정하지 않는다.

## 중단·철회 기준

데모가 연속 리허설에서 정한 안정성 기준을 만족하지 못하거나 복구 시간이 허용 시간을 넘거나 Validator 결과가 재현되지 않으면 Live Demo를 철회하고 정적 Trace 또는 데모 없음으로 전환한다. 구체 기준은 발표 시간과 환경 확인 후 기록한다.
