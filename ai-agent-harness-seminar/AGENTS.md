# AI Agent Harness Seminar 작업 규칙

이 디렉터리와 그 하위 파일을 수정하는 Coding Agent는 다음을 따른다.

1. 작업 전에 Git 상태, 파일 구조, 패키지 관리자와 기존 설정을 실제로 확인한다.
2. docs/SEMINAR_CONTEXT.md를 프로젝트의 Canonical Context로 취급한다. 다른 문서와 충돌하면 임의로 통합하지 말고 충돌을 보고한다.
3. 확인된 사실, 잠정 가정, 해석, 권고를 명시적으로 구분한다.
4. 기술적 사실은 공식 문서나 원출처를 우선하며 URL, 문서명, 확인 날짜를 기록한다.
5. 검증할 수 없는 수치나 효과를 만들지 않는다. Agent가 생산성을 높인다는 주장도 근거 없이 사용하지 않는다.
6. 회사 내부 정보, 실제 업무 데이터, 내부 주소, 계정, 토큰, API Key, 비공개 코드와 실제 로그를 포함하지 않는다. 데모에는 합성 데이터만 사용한다.
7. 발표자료는 한국어 중심으로 작성하되 번역하면 의미가 약해지는 기술 용어는 영어를 유지할 수 있다. 같은 용어는 문서 전체에서 일관되게 쓴다.
8. 새로운 기능이나 의존성을 추가하기 전에 현재 요청 범위를 확인한다. Do Nothing, Chat, Script, Workflow 대안도 필요한 판단에서 유지한다.
9. 가능한 검증 명령을 실제로 실행한 뒤 완료를 보고한다.
10. 결과 보고에서 변경 내용, 검증 내용, 미해결 사항을 구분한다.
11. 이 디렉터리에서 git init을 실행하거나 중첩 저장소·Submodule을 만들지 않는다.
12. Commit과 Push는 명시적 요청 전까지 수행하지 않는다.
