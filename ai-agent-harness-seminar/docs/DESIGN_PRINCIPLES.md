# 디자인 원칙

## 우선순위

장식보다 구조와 정보 전달을 우선한다. 각 슬라이드는 하나의 판단 또는 질문만 남기며, 세부 설명은 Presenter Note와 Handout으로 분리한다.

## 대형 화면 가독성

- 발표 장소 뒤쪽에서 제목·핵심 문장·축 레이블을 읽을 수 있는지 실제 화면으로 확인한다.
- 한 슬라이드의 본문은 핵심 문장과 짧은 비교로 제한한다.
- 표는 열과 행을 줄이고 자세한 항목은 Handout으로 이동한다.
- 화면 해상도와 관람 거리는 미확정이므로 절대 최소 글자 크기는 아직 확정하지 않는다.

## 텍스트와 계층

- 제목은 질문 또는 결론 후보, 본문은 근거, 하단은 출처/상태로 역할을 분리한다.
- 제목 1개, 핵심 문장 1개, 보조 구조 1개를 기본으로 한다.
- 긴 문단, 6개가 넘는 동등 수준 bullet, 문서 전체를 슬라이드에 복사하는 방식을 피한다.
- 코드 블록은 8~12줄 안팎의 핵심만 보이고 파일명·언어·강조 줄을 명시한다. 전체 코드는 저장소에서 연다.

## 한국어와 영어

설명 문장은 한국어 중심으로 쓴다. Agent, Harness, Workflow, Tool, Context, State, Artifact, Validator, Trace, Guardrail, Sandbox, Approval Gate처럼 번역 시 경계가 흐려지는 용어는 영어를 유지하고 첫 등장 때 작업 정의를 제공한다. 같은 개념의 표기를 임의로 바꾸지 않는다.

## 다이어그램

- Mermaid는 7개 이하의 주요 노드와 한 방향 흐름을 기본으로 한다.
- 교차선과 긴 문장을 피하고, 발표 화면에서 레이블이 읽히는지 빌드 후 확인한다.
- Agent Loop는 Observe → Decide → Act → Verify의 폐루프로 그린다.
- Permission, Sandbox, Trace는 단일 단계가 아니라 전체 루프를 둘러싼 Harness 경계로 표현한다.
- 엔지니어링 폐루프 대응은 1:1 비유가 아님을 같은 슬라이드 또는 바로 다음 설명에서 밝힌다.

## 강조

강조 색, 굵기, 위치 중 두 가지 이상을 함께 사용하되 한 화면의 강조 대상은 1~2개로 제한한다. 빨강은 위험·차단, 초록은 검증 성공처럼 의미를 고정하지만 색만으로 상태를 구분하지 않고 아이콘·레이블·선 형태를 함께 사용한다.

## 애니메이션

단계적 흐름을 설명할 때만 최소 click reveal을 쓴다. 장식 전환, 자동 재생, 핵심 비교를 숨기는 애니메이션은 사용하지 않는다. 정적 빌드나 캡처에서도 의미가 유지되어야 한다.

## Theme 선택

| 기준 | Light Theme | Dark Theme |
| --- | --- | --- |
| 밝은 회의실·인쇄 | 반사 환경에서 본문 식별이 쉬울 수 있음 | 배경이 흐리게 보일 수 있음 |
| 어두운 강당 | 화면 눈부심이 클 수 있음 | 집중과 대비에 유리할 수 있음 |
| 코드·그래프 | 선 색상 재조정 필요 | 일부 코드 테마와 자연스러움 |

**잠정 권고:** 초기 샘플은 기본 Light Theme으로 구현한다. 발표 장소 조도, 프로젝터 대비, 영상 촬영 조건을 실제 확인한 뒤 유지하거나 Dark Theme으로 전환한다. 현재 발표 환경은 **알 수 없음**이다.

## 색상과 접근성

- 본문과 배경은 WCAG 대비 검사를 개발 중 참고하되, 프로젝터 실측을 별도로 한다.
- 색상만으로 성공·실패·승인을 표현하지 않는다.
- 그래프는 선 모양, marker, 직접 레이블을 함께 사용한다.
- 붉은색/녹색 조합만 사용하는 비교를 피한다.

## 반복 시각 언어

- Fact: 파란 테두리 + “확인된 사실”
- Assumption: 호박색 점선 + “잠정 가정”
- Recommendation: 보라색 테두리 + “권고”
- Blocked/Approval: 붉은 stop 표식 + 명시적 동사
- Validator success: 체크 표식 + 검사명

기술 용어는 동일한 영문 레이블과 색·도형을 전체 자료에서 반복한다.

## 검증과 철회 조건

1280×720 이상 발표 화면과 실제 프로젝터에서 제목·본문·다이어그램을 확인한다. 뒤쪽 좌석에서 읽기 어렵거나 색 구분이 사라지거나 Mermaid 레이블이 겹치면 해당 레이아웃을 폐기한다. 원칙은 2026-07-21 초안이며 발표 장소 확인 후 재검토한다.
